What is a Data Structure?

A data structure is a method of organizing and storing data in a computer so that it can be used efficiently. The choice of data structure affects both time complexity and space complexity of operations such as searching, insertion, deletion, and retrieval. :contentReference[oaicite:1]{index=1}

  • Helps to store large data in a structured way
  • Improves performance of algorithms
  • Supports operations like processing, retrieving, and updating data

Need for Data Structures

We use data structures because:

  • They make data representation clear and easy to understand.
  • They allow fast access and modification of data.
  • They help reduce memory usage and execution time.
  • They provide systematic ways to organize, manage, and store data.

Basic Terminology

  • Data: Raw values or facts.
  • Information: Organized data that has meaning and can be used for decisions.
  • Entity: A real-world object (e.g., student) with attributes like name, age, etc.
  • Field: Single attribute of an entity (e.g., roll number).
  • Record: Collection of fields describing one entity.
  • File: Collection of records of the same type.
  • Key / Primary Key: Field whose value uniquely identifies a record.

Classification of Data Structures

Data structures are broadly classified into two categories:

Primitive Data Structures

Basic data types supported directly by the hardware or programming language.

  • Integer
  • Float / Double
  • Character
  • Boolean
  • String
  • Pointers

Non-Primitive Data Structures

Derived from primitive types and used to represent complex data.

  • Linear: Array, Linked List, Stack, Queue
  • Non-Linear: Trees, Graphs

Linear vs Non-Linear Data Structures

Feature Linear Non-Linear
Arrangement of Data Elements are stored sequentially, one after another. Elements are stored in hierarchical or network form.
Traversal Can often be traversed in a single pass. May require multiple passes and different traversal strategies.
Examples Array, Linked List, Stack, Queue Tree, Graph
Typical Use General programming, simple storage and processing. AI, routing, image processing, hierarchical data.

Quick Summary

Primitive data types are building blocks. Non-primitive data structures combine these primitives to handle larger and more complex datasets efficiently. Choosing the right structure for the right problem is a key part of algorithm design.
